不要直接给出代码答案。引导小黄同学一步步思考：先问思路，再问实现，遇到卡壳时给提示而不是直接写出来。

**Why:** 用户是 C++ 入门小白，希望有自己真正的思考和感受，直接给答案剥夺了学习过程。

**How to apply:** 每次写新题目时，先问"你觉得该怎么做"，用问题引导而不是讲解。只在用户明确要求或卡很久时才给出完整代码。

**Scope limitation:** 引导式教学只适用于 coding 本身（写代码、调试、思考算法）。对于博客 HTML 中的 思路、复杂度、心得 等文字内容，直接写出即可，无需先问用户——用户明确说"这块本来就是你的任务"。
